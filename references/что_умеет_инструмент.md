# Что Умеет Этот Инструмент

`TwitterAPI X Reader` нужен для тех случаев, когда надо быстро и стабильно получить данные из X (Twitter), не упираясь в капризы самого `x.com`: логин, JavaScript, региональные ограничения, пустые страницы или неудобное отображение.

Проще говоря, это инструмент для нормального чтения и, при необходимости, аккуратной работы с контентом из X через API.

## Что он умеет

Инструмент умеет:

- доставать обычные твиты по ссылке или по id;
- открывать длинные статьи X Article, даже если в браузере они отображаются нестабильно;
- искать пользователей и получать информацию о них;
- смотреть, что публикует аккаунт, кому он отвечает, кто на него подписан и на кого подписан он сам;
- получать связанные данные по твиту: ответы, ретвиты, цитаты, лайки и другие реакции;
- работать с подборками и сущностями вокруг X: списки, сообщества, Spaces, тренды;
- обращаться к служебным endpoint'ам аккаунта, если это потребуется в работе;
- в явном режиме выполнять действия от имени аккаунта, если пользователь сам этого попросил.

## Чем он особенно полезен

Инструмент особенно полезен, когда нужно:

- быстро разобрать статью или тред и вытащить из них смысл;
- проверить конкретный твит, не открывая руками X;
- собрать контекст по автору, теме или обсуждению;
- использовать данные из X как вход для анализа, исследований или продуктовых выводов;
- опираться не на скриншоты и пересказы, а на структурированные данные.

## Что он делает лучше обычного открытия X

Когда ты просто открываешь ссылку на `x.com`, часто возникают проблемы:

- страница не грузится без авторизации;
- статья видна у человека в приложении, но не видна в браузере;
- сложно быстро вытащить текст, метаданные и структуру;
- неудобно использовать это дальше в аналитике.

Этот инструмент решает именно эту проблему: он получает данные напрямую через API и возвращает их в понятном, пригодном для работы виде.

## Что он умеет уже сейчас в нашей сборке

В текущем виде у нас есть два режима работы:

1. Узкий и удобный режим для частой задачи:
   `достань твит или статью по ссылке`

2. Универсальный режим:
   `обратись к любому поддерживаемому методу из документации twitterapi.io`

Это значит, что инструмент не ограничен только статьями. Он уже подготовлен под более широкий набор задач, если они понадобятся позже.

## Важное ограничение

Инструмент не должен без необходимости совершать действия от имени аккаунта.

По умолчанию он работает как инструмент чтения и анализа. Если API позволяет что-то менять, например:

- опубликовать пост;
- лайкнуть;
- ретвитнуть;
- подписаться;
- добавить в закладки;

то такие действия допустимы только по прямому запросу пользователя.

## Как Вызвать Инструмент

Есть три нормальных способа вызвать этот инструмент.

Важно: короткие команды `xread` и `xapi` сделаны как переносимые пользовательские команды. Они не должны зависеть от конкретного проекта и работают через personal skill в `~/.codex/skills/twitterapi_x_reader`.

### 1. Самый простой способ: через skill

Если ты работаешь в Codex и хочешь просто попросить инструмент сделать работу за тебя, можно вызывать его так:

- `$twitterapi_x_reader`

Примеры нормальных запросов:

- `$twitterapi_x_reader достань статью по этой ссылке и кратко перескажи`
- `$twitterapi_x_reader проверь твит и скажи, есть ли там что-то полезное для нашего проекта`
- `$twitterapi_x_reader собери данные по автору и его последним постам`

Это лучший вариант, когда тебе нужен не JSON, а готовый результат: summary, выводы, разбор, сравнение или аналитика.

### 2. Прямой вызов для твита или статьи

Если нужно просто достать содержимое по ссылке или по id, можно вызывать специальную команду:

```bash
xread "https://x.com/ZenithTON/status/2046570503801119055"
```

Можно передать и просто id твита:

```bash
xread "2046570503801119055"
```

Если нужно явно сказать, что ты хочешь получить именно статью или именно твит:

```bash
xread "2046570503801119055" --mode article
xread "2046570503801119055" --mode tweet
```

Если по какой-то причине короткая команда недоступна, остаётся полный прямой вызов:

```bash
python3 ~/.codex/skills/twitterapi_x_reader/scripts/twitterapi_fetch.py --input "https://x.com/ZenithTON/status/2046570503801119055"
```

### 3. Универсальный вызов для любого метода API

Если нужен не только твит или статья, а любой другой поддерживаемый метод `twitterapi.io`, используется универсальная команда:

```bash
xapi --method GET --path /oapi/my/info --query-json '{}'
```

Если нужен полный прямой вызов без короткой команды:

```bash
python3 ~/.codex/skills/twitterapi_x_reader/scripts/twitterapi_call.py --method GET --path /oapi/my/info --query-json '{}'
```

Через неё можно обращаться к любым поддерживаемым endpoint'ам из документации, если тебе нужны:

- данные аккаунта;
- поиск;
- пользователи;
- подписчики и подписки;
- списки, сообщества, Spaces, тренды;
- другие чтения через API.

## Какой способ выбирать

Если коротко:

- если тебе нужен готовый человеческий результат, вызывай `$twitterapi_x_reader`;
- если нужно быстро вытащить конкретный твит или статью, используй `xread`;
- если нужен более редкий или специфический метод из документации, используй `xapi`.

## Как Обновить Инструмент

Если ты доработал skill в репозитории и хочешь обновить переносимую версию у себя на машине, используй одну команду из корня репозитория:

```bash
./install_portable.sh
```

Что делает эта команда:

- обновляет personal skill в `~/.codex/skills/twitterapi_x_reader`;
- переустанавливает глобальные команды `xread` и `xapi` в `~/.local/bin`;
- сохраняет локальный файл с API key, если он уже существует.

Это и есть основной способ поддерживать portable-версию в актуальном состоянии.

## Какой от него практический смысл

Инструмент даёт три практических преимущества:

- позволяет быстро разбирать материалы из X без ручной возни;
- снижает риск, что важный пост или статья окажутся "невидимыми" из-за интерфейса X;
- делает X нормальным источником данных для исследований, продуктовых гипотез и анализа рынка.

Если коротко: это не просто "парсер твиттера", а рабочий мост между X и исследовательскими задачами.
