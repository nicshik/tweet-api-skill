# Что Умеет Этот Инструмент

`TwitterAPI X Reader` нужен в тех случаях, когда надо быстро и стабильно получить данные из X, не упираясь в капризы самого `x.com`: логин, JavaScript, пустые страницы, странное отображение статей или неудобный ручной разбор.

Проще говоря, это рабочий мост между X и нормальной исследовательской работой: прочитать пост, вытащить статью, посмотреть автора, собрать контекст по обсуждению и уже потом делать выводы.

## Что он умеет

Инструмент умеет:

- доставать обычные твиты по ссылке или по id;
- открывать длинные X Articles;
- получать профиль автора;
- забирать последние посты аккаунта;
- смотреть ответы на твит;
- смотреть quote tweets;
- искать аккаунты по теме;
- искать твиты по ключевым словам и формулировкам;
- обращаться и к другим методам `twitterapi.io`, если они есть в документации.

## Когда он особенно полезен

Он полезен, когда нужно:

- быстро разобрать статью или тред и понять, есть ли там смысл для проекта;
- проверить конкретный пост, не открывая X вручную;
- собрать контекст по автору и его последним публикациям;
- понять реакцию на пост через ответы и цитаты;
- использовать данные из X как вход для продуктового, конкурентного или рыночного анализа.

## Как его вызывать

Есть три нормальных способа работы.

### 1. Через skill в агенте

Если ты работаешь в среде, где skill вызывается по имени, можно формулировать задачу прямо словами:

- `$twitterapi_x_reader достань статью по ссылке и кратко перескажи`
- `$twitterapi_x_reader проверь твит и скажи, есть ли там что-то полезное для проекта`
- `$twitterapi_x_reader собери контекст по автору и его последним постам`

Это лучший вариант, когда тебе нужен не сырой JSON, а уже готовый разбор.

### 2. Через `xread`

Это самый простой рабочий способ, когда нужно быстро получить сам твит или статью.

Пример по ссылке:

```bash
xread "https://x.com/ZenithTON/status/2046570503801119055"
```

Пример по id:

```bash
xread "2046570503801119055"
```

Если надо явно сказать, что ты хочешь именно статью:

```bash
xread "2046570503801119055" --mode article
```

Если по какой-то причине короткая команда недоступна, остаётся прямой вызов скрипта:

```bash
python3 ~/.codex/skills/twitterapi_x_reader/scripts/twitterapi_fetch.py --input "https://x.com/ZenithTON/status/2046570503801119055"
```

### 3. Через `xapi`

Это универсальный режим для любого поддерживаемого метода `twitterapi.io`.

Пример:

```bash
xapi --method GET --path /oapi/my/info --query-json '{}'
```

Если нужен прямой вызов без короткой команды:

```bash
python3 ~/.codex/skills/twitterapi_x_reader/scripts/twitterapi_call.py --method GET --path /oapi/my/info --query-json '{}'
```

## Готовые примеры для ежедневной работы

### Прочитать статью или твит

```bash
xread "https://x.com/ZenithTON/status/2046570503801119055"
```

### Принудительно читать как статью

```bash
xread "2046570503801119055" --mode article
```

### Посмотреть профиль автора

```bash
xapi --method GET --path /twitter/user/info --query-json '{"userName":"ZenithTON"}'
```

### Посмотреть последние посты автора

```bash
xapi --method GET --path /twitter/user/last_tweets --query-json '{"userName":"ZenithTON","includeReplies":false}'
```

### Посмотреть ответы на конкретный пост

```bash
xapi --method GET --path /twitter/tweet/replies/v2 --query-json '{"tweetId":"2046570503801119055","queryType":"Latest"}'
```

### Посмотреть quote tweets

```bash
xapi --method GET --path /twitter/tweet/quotes --query-json '{"tweetId":"2046570503801119055","includeReplies":false}'
```

### Найти аккаунты по теме

```bash
xapi --method GET --path /twitter/user/search --query-json '{"query":"TON AI"}'
```

### Найти твиты по теме

```bash
xapi --method GET --path /twitter/tweet/advanced_search --query-json '{"query":"\"AI agents\" Telegram TON","queryType":"Top"}'
```

## Какой способ выбирать

Если коротко:

- если нужен готовый человеческий разбор, вызывай skill;
- если нужно быстро достать конкретный твит или статью, используй `xread`;
- если нужен более редкий или точный метод из документации, используй `xapi`.

## Как работать из Windsurf и Antigravity

Практический смысл одинаковый: агент должен понимать, что у тебя есть готовые команды `xread` и `xapi`.

### В Windsurf

Если в проекте есть `AGENTS.md`, туда стоит явно записать правило вроде:

- при работе с X сначала использовать `xread` и `xapi`, а не пытаться открывать `x.com` напрямую;
- для чтения поста использовать `xread`;
- для остальных методов `twitterapi.io` использовать `xapi`.

Тогда запросы можно формулировать так:

- `посмотри этот твит через xread и скажи, есть ли там полезные идеи`
- `через xapi собери профиль автора и последние посты`
- `проверь ответы и цитаты к этому посту`

### В Antigravity

Логика та же самая: если агент видит локальные команды `xread` и `xapi`, он может использовать их как основной путь работы с X.

Примеры нормальных запросов:

- `используй xread и достань содержимое статьи`
- `используй xapi и найди последние посты этого аккаунта`
- `используй xapi и поищи обсуждения темы AI agents в контексте Telegram и TON`

## Как обновить инструмент

Если ты доработал репозиторий и хочешь обновить portable-версию на своей машине, из корня репозитория запускается:

```bash
./install_portable.sh
```

Эта команда:

- обновляет skill в `~/.codex/skills/twitterapi_x_reader`;
- обновляет глобальные команды `xread` и `xapi` в `~/.local/bin`;
- не трогает локальный файл с API key.

## Важное ограничение

По умолчанию инструмент должен использоваться в read-only режиме.

Если API позволяет что-то менять от имени аккаунта, такие действия допустимы только по прямой и явной просьбе пользователя.

## Практический смысл

Если совсем коротко: это не просто "парсер твиттера", а удобный рабочий слой, который превращает X в нормальный источник данных для анализа, исследований и продуктовых выводов.

Если нужны ещё более прикладные сценарии, они отдельно собраны в `references/research_examples.md`.
